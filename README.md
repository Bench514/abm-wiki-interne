# Exercice GitHub travail d'équipe

### Contexte:

Atelier Boréal Médias Inc. (ABM) n'a pas encore de manuel interne. Les procédures, politiques et informations sur l'équipe TI existent dans des dizaines de fichiers éparpillés sur des postes différents. Le résultat : personne ne sait quelle version est la bonne.


---
### Mission:

Contribuer à la création d'un wiki interne versionné pour ABM. Ton équipe va construire ce wiki ensemble, en parallèle, en utilisant Git et GitHub.

Chaque membre de l'équipe est responsable d'une section du wiki. À la fin de la séance, toutes les contributions seront réunies dans un seul dépôt cohérent, avec un historique complet de qui a fait quoi.

En premier, tu dois:
1. Recherchez sur le web des exemples de documentation pour la section de WIKI sur laquelle tu travailles
2. Reproduir cette documentation dans le fichier `.md` désigné.


#### Ce que tu vas pratiquer :

- Tes nouvelles compétences en MarkDown ;-)
- Cloner un dépôt existant
- Créer et gérer une branche de travail
- Faire des commits progressifs et significatifs
- Synchroniser ton travail avec celui de l'équipe
- Ouvrir et réviser une Pull Request
- Résoudre un conflit de version
